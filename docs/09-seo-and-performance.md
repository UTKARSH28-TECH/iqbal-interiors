\# SEO \& PERFORMANCE



\# Purpose



This document defines the SEO, accessibility, and performance standards for the Iqbal Interiors website.



The website should rank well in search engines while delivering a fast and smooth experience on desktop and mobile devices.



\---



\# SEO Goals



The website should:



\* Rank for local interior design searches.

\* Increase organic traffic.

\* Improve Google Business visibility.

\* Generate more WhatsApp enquiries.

\* Increase showroom visits.



\---



\# Metadata



Every page should have:



\* Unique page title

\* Unique meta description

\* Canonical URL

\* Open Graph image

\* Twitter Card metadata



Titles should remain under 60 characters.



Descriptions should remain under 160 characters.



\---



\# Page Titles



Examples:



Home



Iqbal Interiors | Premium Interior Design in Giridih



Bedroom



Bedroom Interior Designs | Iqbal Interiors



Kitchen



Modular Kitchen Designs | Iqbal Interiors



False Ceiling



Modern False Ceiling Designs | Iqbal Interiors



Contact



Contact Iqbal Interiors | Jamua, Giridih



\---



\# Open Graph



Every page should generate:



\* og:title

\* og:description

\* og:image

\* og:url

\* og:type



Shared WhatsApp links should display attractive previews.



\---



\# Structured Data



Implement Schema.org structured data.



Recommended schemas:



\* Local Business

\* Organization

\* WebSite

\* Image Gallery

\* BreadcrumbList



Future support:



\* Review

\* Product

\* Article



\---



\# Images



All images should:



\* Use Next.js Image component

\* Include descriptive alt text

\* Use responsive sizing

\* Be lazy loaded

\* Prefer WebP

\* Reserve layout space to avoid CLS



Hero images may use priority loading.



\---



\# Core Web Vitals



Target scores:



Largest Contentful Paint (LCP)



Less than 2.5 seconds



Interaction to Next Paint (INP)



Less than 200 ms



Cumulative Layout Shift (CLS)



Less than 0.1



\---



\# Accessibility



Requirements:



\* Semantic HTML

\* Keyboard navigation

\* Visible focus indicators

\* Proper heading hierarchy

\* Accessible forms

\* ARIA labels where appropriate

\* High color contrast

\* Screen reader friendly



Target WCAG 2.1 AA compliance.



\---



\# Mobile Performance



Prioritize:



\* Fast loading

\* Optimized images

\* Smooth scrolling

\* Touch-friendly controls

\* Lightweight animations



The mobile experience should be considered the primary experience.



\---



\# Lazy Loading



Apply lazy loading to:



\* Gallery images

\* Below-the-fold sections

\* Non-critical assets



Load only what is necessary initially.



\---



\# Performance Optimization



Use:



\* Static rendering where possible

\* Code splitting

\* Dynamic imports for heavy components

\* Image optimization

\* Font optimization

\* Tree shaking



Avoid unnecessary JavaScript.



\---



\# URL Structure



Keep URLs simple and descriptive.



Examples:



/



/gallery



/gallery/bedroom



/gallery/kitchen



/services



/about



/contact



Avoid query parameters for permanent pages.



\---



\# Internal Linking



Encourage exploration by linking:



\* Related gallery categories

\* Services

\* About page

\* Contact page



This improves both SEO and user engagement.



\---



\# Social Sharing



Gallery pages should generate rich previews when shared on:



\* WhatsApp

\* Facebook

\* LinkedIn



Each shared link should include:



\* Cover image

\* Category title

\* Short description



\---



\# Future SEO



The website should support:



\* Blog articles

\* Interior design tips

\* Project case studies

\* FAQs

\* Video content



without requiring major architectural changes.



\---



\# Monitoring



Future integrations may include:



\* Google Analytics

\* Google Search Console

\* Microsoft Clarity

\* Meta Pixel



These should be easy to add later.



\---



\# Final Goal



The website should achieve:



\* Excellent Lighthouse scores

\* Strong Core Web Vitals

\* Fast page loads

\* High accessibility

\* Search engine visibility

\* Great user experience on all devices

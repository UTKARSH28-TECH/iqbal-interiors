\# DEVELOPMENT ROADMAP



\# Purpose



This document defines the development workflow for the Iqbal Interiors website.



Claude should follow this roadmap throughout the project.



The objective is to build a production-ready, maintainable, scalable website incrementally, validating each stage before moving to the next.



\---



\# General Principles



Before writing any code:



\* Read every document inside the `docs/` folder.

\* Review the `design-inspiration/` folder for UI references.

\* Review the `public/images/` folder to understand available assets.

\* Never assume business information that has not been documented.

\* Ask for clarification when required.



\---



\# Development Order



The project should be built in the following sequence.



\## Phase 1 — Foundation



\* Configure project structure

\* Global layout

\* Theme

\* Typography

\* Colors

\* Metadata

\* Utility functions

\* Image helpers



No homepage sections should be built yet.



\---



\## Phase 2 — Navigation



Build:



\* Desktop Navbar

\* Mobile Navigation

\* Sticky Header

\* Responsive Menu

\* WhatsApp Button



Wait for approval.



\---



\## Phase 3 — Hero Section



Build:



\* Hero slider

\* Premium typography

\* CTA buttons

\* Smooth transitions

\* Responsive layout



Use only real project photography.



Wait for approval.



\---



\## Phase 4 — Homepage Sections



Build one section at a time:



\* Trust Statistics

\* Services

\* Gallery Categories

\* Featured Projects

\* Why Choose Us

\* Showroom Preview

\* Owner Section

\* Contact CTA

\* Footer



Pause after each major section for review.



\---



\## Phase 5 — Gallery



Implement:



\* Dynamic gallery architecture

\* Category pages

\* Responsive grid

\* Lightbox

\* Shareable URLs

\* Related categories



Gallery must support future expansion without code changes.



\---



\## Phase 6 — Additional Pages



Build:



\* About

\* Services

\* Contact



Use reusable components wherever possible.



\---



\## Phase 7 — Optimization



Review:



\* Accessibility

\* SEO

\* Performance

\* Responsiveness

\* Animations

\* Image loading



Resolve issues before deployment.



\---



\# Coding Workflow



For every task:



1\. Explain the plan.

2\. Implement the feature.

3\. Summarize the changes.

4\. Wait for approval before continuing.



Do not implement multiple major features without confirmation.



\---



\# Git Workflow



Use focused commits.



Examples:



\* feat: build responsive navbar

\* feat: add hero slider

\* feat: implement gallery grid

\* fix: improve mobile spacing

\* refactor: extract reusable gallery card



\---



\# Quality Checklist



Every completed feature must be:



\* Responsive

\* Accessible

\* Type-safe

\* Performance optimized

\* Consistent with the design system

\* Built with reusable components

\* Free of console errors



\---



\# Future Expansion



The architecture should support future additions such as:



\* Blog

\* Video Gallery

\* Reviews

\* Appointment Booking

\* Multi-language support

\* Admin Dashboard

\* CMS integration



without major restructuring.



\---



\# Final Objective



The finished website should not feel like an AI-generated template.



It should feel like a custom-built premium website that showcases Iqbal Interiors' craftsmanship, encourages enquiries, and remains easy to maintain as the business grows.
